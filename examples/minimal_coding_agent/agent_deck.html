Solve Flow

What Happens During agent.py solve

BPMN topics
  • initialize
  • analyze_error
  • select_strategy
  • apply_fix
  • run_tests_after
  • log_attempt / increment_retry / write_report
Runtime behavior
  • Runs tests, identifies failure class.
  • Tries LLM repair or deterministic fallback.
  • Loops via gateway until success or retry limit.
  • Writes machine-readable report with run_id.

Tradeoffs

Where Off-The-Shelf Agents Still Win Today

They are stronger on
  • Large-repo contextual understanding.
  • Long-horizon autonomous editing.
  • General coding task breadth and speed.
  • Polished integrated developer UX.
SPEAR can close gap via roadmap
  • Planner loop upgrades in BPMN.
  • AST-aware patching and retrieval.
  • Expanded benchmark suites and CI trends.
  • Human-approval and policy guardrails.
Capability maturity is an explicit engineering plan
Positioning

Recommended Message For Stakeholders

Say this

"This is an explainable coding workflow orchestrated by SPEAR's BPMN-RDF engine, with run-level provenance and deterministic fallback."

Do not say this

"This is a full replacement for production-grade coding agents."

Showcase KPICan we explain every decision and artifact by run_id?
Maturity KPICan we improve pass-rate trend in evals/trend.csv over releases?
Precise positioning builds trust
